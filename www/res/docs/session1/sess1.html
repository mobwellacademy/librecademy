Java is the syntax choosen to define the behaviour of your program

Java for Android is not the same J2SE you know: it's a different virtual machine (DalvikVM) optimized to work with the Android operating system, and with low resource devices in general. Most of the time you wont notice the difference.

XML is the syntax choosen to define the resources you use

Example

  • colors.xml: Where your the colors you use are defined.

    In Java You'll use like this 'getResources().getColor(R.color.idname);'
    <item name="blue" type="color">#FF33B5E5</item>

  • Latter on, you'll see how this slide is missing so much detail, like defining layouts for you Views, for example

Gradle is the tool you'll want to use to build your magnificient .APK

Yes, you can tell the compiler where your project files are and how they interact, but I suppose you won't do this, for mental sanity reasons of course.

Trust me on this one: Android Studio does such an amazing job generating and managing .gradle files, that you can build a more than a thousand downloads app without even knowing it exists.
